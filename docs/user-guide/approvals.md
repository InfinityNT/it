# Approvals

The approvals system adds a review step to device operations, ensuring that assignments and other actions are authorized before they take effect.

## How Approvals Work

When the approval workflow is active:

1. A user initiates an action (e.g., assigning a device)
2. The action creates an approval request instead of executing immediately
3. An authorized approver reviews the request
4. The approver either **approves** or **rejects** the request
5. If approved, the original action is executed

## Viewing Approvals

The Approvals page shows:

- **Pending** — Requests waiting for review
- **Approved** — Recently approved requests
- **Rejected** — Requests that were denied

Each approval entry shows the requester, the action requested, the target device/employee, and the current status.

## Approving or Rejecting

If you have approval permissions:

1. Open the Approvals page
2. Review the pending request details
3. Click "Approve" or "Reject"
4. Add an optional note explaining your decision

## Approval Notifications

When an approval is submitted, approved, or rejected, the relevant users will see the status change reflected in the activity feed and the approvals list.
