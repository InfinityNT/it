# Assignments

Assignments track which devices are issued to which employees. This section covers how to view, create, and manage device assignments.

## Assignment List

The assignments page shows all active and historical assignments with:

- **Device** — The assigned device's asset tag and model
- **Employee** — The person the device is assigned to
- **Assigned Date** — When the assignment was created
- **Status** — Active, Returned, or Pending Approval

## Assigning a Device

To assign a device to an employee:

1. Click "Assign Device" from the assignments page or a device's detail view
2. Select the device (only available devices are shown)
3. Select the employee
4. Add any notes about the assignment
5. Submit the form

If the approval workflow is enabled, the assignment will go into a "Pending Approval" state until an authorized user approves it.

## Returning a Device

To return a device:

1. Find the active assignment in the list
2. Click the "Return" action
3. Confirm the return and add any return notes
4. The device status will be updated to "Available"

## Assignment History

Each device and employee has a full assignment history visible in their detail views, showing all past and current assignments with dates and notes.
